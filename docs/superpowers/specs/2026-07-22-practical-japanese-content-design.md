# 実務日本語化・全ページ内容刷新 設計仕様書

## 1. 目的

Finance Modeling Labの全ページとダウンロード用Excelを、国内の事業会社、金融機関、M&Aアドバイザリー、投資部門で自然に読める表現へ統一する。単純な英日置換ではなく、読者が資料収集、Excel作成、レビュー、意思決定まで再現できる実務教材へ内容を刷新する。

編集名義は既存方針どおり「Finance Modeling Lab 編集部」とする。

## 4. 各ページの実務教材構成

主要な解説ページは、可能な限り次の順序で構成する。

1. このページで作る成果物
2. 利用する業務場面と想定読者
3. 必要資料、基準日、情報源
4. Excelのシート構成と入力項目
5. 計算手順とセル式
6. 具体的な数値例

ページの冒頭では定義説明に終始せず、「業務で何を完成させるか」を先に示す。説明は架空の数値例であることを明記し、実在企業の事実であるかのような表現をしない。

## 5. 分野別の内容要件

### 5.1 会計仕訳

仕訳例は具体的な金額を置き、借方・貸方、決算整理前後、PL・BS・CFへの影響を一つの流れで示す。最低限、売上計上、売掛金回収、仕入・買掛金、減価償却、未払費用、前払費用、引当金、税金、借入・利息を扱う。

例は次の粒度で記載する。

- 取引：12月分の外注費1,200千円を未払計上
- 決算仕訳：借方 外注費1,200千円 / 貸方 未払費用1,200千円
- PL：営業利益が1,200千円減少
- BS：未払費用が1,200千円増加
- CF：当期未払いのため直接法・間接法の扱いを注記
- モデル：費用発生と支払時期を分け、未払費用残高と運転資本増減へ反映

### 5.2 財務三表モデル

入力、前提、PL、BS、CF、計算明細、チェックの役割を示す。売上から営業利益、運転資本、設備投資、借入、現金残高までを連動させ、BS貸借一致と現金増減一致を必須チェックとする。

### 5.3 ValuationとDCF

非上場会社を想定し、事業計画の正常化、WACC、FCFF、継続価値、Enterprise Value、Equity Valueを一連の作業として説明する。

- WACCでは、類似上場会社のβ、資本構成、負債コスト、実効税率の設定根拠を説明する。
- FCFFでは、EBIT、税金、減価償却、設備投資、運転資本増減の符号を具体例で示す。
- 継続価値では、永久成長率の根拠とWACCを下回る必要性を説明する。
- Enterprise ValueからEquity Valueへの調整では、現金及び現金同等物、有利子負債、有利子負債類似項目、非支配持分の基準日と符号を明記する。
- Base / Upside / DownsideとWACC・永久成長率の感応度を混同しない。

### 5.4 類似会社比較

比較会社は、業種名だけでなく、収益構造、顧客、地域、事業構成、規模、成長率、利益率、資本集約度、会計方針、開示の質で選定する。

比較候補会社一覧から主要比較会社を選ぶ過程、除外理由、補完比較会社の使い方、採用マルチプルのレンジを説明する。単純な合計点や中央値を正解として扱わず、重大な不一致と情報不足を独立して判定する。

## 6. Excel設計

### 6.1 DCF評価モデル

シート名は次に統一する。

`表紙 / 入力 / 前提 / PL / BS / CF / 計算明細 / DCF / チェック`

青セルを入力、黒文字を計算式、緑文字または緑の参照表示を他シート参照、強調枠を主要出力とする。各シート上部には、目的、入力者が更新するセル、基準日、単位、確認事項を記載する。

### 6.2 類似会社選定ワークシート

シート名は次に統一する。

`使い方 / 対象会社 / 比較候補会社一覧 / 比較会社選定表 / 比較上の位置づけ / 検討記録 / チェック`

保存する内部コードは数式の安定性のため英語でもよいが、利用者に見せる選択肢と説明は日本語にする。Web上の図、Excelのシート名、検証式は同じ名称を参照する。

### 6.3 配布互換性

新しい日本語ファイル名をダウンロード時の推奨名とする。既存URLからのリンク切れを防ぐ必要がある場合は旧ファイルを互換用に残す。Web上の説明、更新日、容量は実ファイルと一致させる。

## 7. 画面設計

- 装飾だけの英語小見出しは、内容を表す日本語へ変更する。
- 表はPCで一覧性を保ち、スマートフォンでは横スクロール可能にする。
- ワークシート図はExcelの行番号、列見出し、入力色、式、出力値と一致させる。
- 長いページにはページ内目次、レビュー項目、次の学習導線を置く。
- ダウンロードボタンにはファイル内容、対象者、更新日、利用条件を明記する。

## 8. 検索とURL互換性

既存URLは変更しない。本文は実務的な日本語を優先し、旧英語用語は検索用キーワード、メタデータ、同義語説明として必要最小限に保持する。ページタイトルと説明文は、日本語の検索意図と成果物が分かる内容へ修正する。

検索機能では、日本語の新用語と従来の英語用語のどちらでも該当ページへ到達できるようにする。

- GitHub Pages公開後、主要ページとダウンロードURLがHTTP 200を返す。
